## What this does

One or two sentences. State the change, not the journey.

## Why

Link the issue: Closes #___ — or explain the motivation if there isn't one.

## Type of change

- feat: new functionality
- fix: bug fix
- docs: documentation only
- refactor: no functional change
- chore: tooling, deps, config
- breaking change

## How this was tested

What you ran, what you checked manually, what's covered by automated tests.

## Checklist

- I read CONTRIBUTING.md
- Tests added or updated for behavior changes
- Lint, typecheck, and test pass locally
- Docs updated if public API or behavior changed
- No breaking change, or it's called out above and documented
